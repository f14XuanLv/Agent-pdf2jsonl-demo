# Agent-pdf2jsonl

`Agent-pdf2jsonl` 是一个高度智能化的数据处理流水线，专门用于从PDF格式的法律文档中提取结构化信息，并将其转换为易于使用的 JSONL (JSON Lines) 格式。该系统利用 Google Gemini 2.5 Flash 的强大能力，通过一个复杂的多阶段代理（Agent）架构，实现了对文档内容的高精度、自动化分析与提取。

## 核心功能

- **多阶段处理流水线**: 系统采用分步执行的策略，将复杂的提取任务分解为模板选择、任务规划、PDF切片、知识提取和智能重试等多个阶段，确保了处理过程的鲁棒性和可控性。
- **动态模板系统**: 内置一套位于 `extraction_templates_v2/` 的精细化提取模板，覆盖了从完整法律、修正案到司法解释、表格、目录等多种常见的法律文书类型。
- **智能模板选择与优化**: **阶段一（PlannerSelector）** 会分析整个PDF的结构，自动选择最匹配的模板，并根据文档的实际层级深度（如编、章、节、条）对模板进行“精确化”修剪，生成一个为当前文档量身定制的提取方案。
- **逻辑任务切分**: **阶段二（PlannerPartitioner）** 负责将PDF文档按逻辑结构（而非固定页数）切分为多个子任务（Chunks）。它会为每个任务定义精确的页码范围和文字描述的提取边界，确保内容提取的完整性和上下文连续性。
- **物理PDF切片**: **阶段2.1（PDFSlicer）** 会根据任务规划，将原始PDF物理切割成多个小PDF文件，每个文件对应一个子任务，这大大降低了后续处理的复杂度。
- **并发提取与错误处理**: **阶段三（Extractor）** 使用线程池并发处理所有子任务，调用Gemini API进行最终的知识提取。系统拥有强大的API请求管理器，包含日志记录、自动重试和基于HTTP状态码（如429, 503）的API密钥动态切换机制。
- **智能重分割**: **阶段2.5（PlannerRepartitioner）** 是一个独特的容错机制。当一个任务因为内容过大或复杂（导致模型输出超限）而失败时，该模块会智能地将其重新分割成两个更小的、逻辑连贯的子任务，并再次尝试处理，极大地提高了处理成功率。
- **状态管理与断点续传**: 系统会自动记录每个PDF的处理状态（已完成、待重试），支持在中断后从上次失败的地方继续处理，非常适合大规模的批量任务。
- **灵活的密钥管理**: `keys.py` 提供了一个线程安全的API密钥管理器，通过加权随机轮询算法来分配密钥，有效规避API速率限制。

## 工作流程

1.  **扫描与初始化**: 代理首先扫描指定的PDF目录（如 `国家级法律条款/`），并创建用于存放输出（`_jsonl`）、PDF切片（`_pdf_slices`）和日志（`log`）的目录结构。
2.  **状态检查**: 加载已处理和待重试的PDF列表，确定本次需要处理的文件。
3.  **单文件处理循环**:
    a.  **Phase 1: 模板规划**: `PlannerSelector` 分析PDF，选择并优化模板。
    b.  **Phase 2: 任务分割**: `PlannerPartitioner` 将PDF的提取任务分解为详细的执行计划。
    c.  **Phase 2.1: 物理切片**: `PDFSlicer` 根据执行计划将PDF切割成小块。
    d.  **Phase 3: 并发提取**: `Extractor` 并发执行所有任务，调用Gemini API提取知识单元。
    e.  **Phase 2.5: 智能重试**: 如果有任务因内容超长失败，`PlannerRepartitioner` 会介入，将其分割并重新执行。
4.  **结果合并**: 所有成功的任务结果被合并，为每个PDF生成一个对应的 `.jsonl` 文件。
5.  **状态更新**: 更新已完成和待重试列表。

## 如何使用

1.  **配置密钥**: 在 `keys.py` 文件中的 `_KEYS` 列表中填入你的Google Gemini API密钥。
2.  **准备数据**: 将需要处理的PDF文件放入一个目录中（例如 `国家级法律条款/`）。
3.  **配置代理**: 在 `agent_v2.py` 的 `if __name__ == '__main__':` 部分，修改 `pdf_directory` 变量，使其指向你的PDF目录。
4.  **运行代理**:
    ```bash
    python agent_v2.py
    ```
5.  **查看结果**: 处理完成后，结构化的数据将以 `.jsonl` 格式保存在 `[pdf_directory_name]_jsonl/` 目录中。
